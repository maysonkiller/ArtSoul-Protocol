# ArtSoul Base/X Progress Report — 2026-07-17 to 2026-07-21

Audience: Base ecosystem and public build-in-public followers.

Status boundary: this report includes only work merged to `main` or completed
operational verification during the reporting window. Moderation access and
public complaint intake are implemented behind disabled flags and are not
described as active production features.

## What shipped

### Infrastructure and security

- Reconciled the Base Sepolia indexer's configured confirmation depth with its
  persisted status without resetting the indexing cursor.
- Made reorg rollback schema-aware so the current projection schema no longer
  attempts to access retired legacy tables.
- Verified the production migration and RLS baseline without manufacturing
  historical ledger entries.
- Applied storage MIME allowlisting and a 50 MB upload ceiling while preserving
  the existing spend cap.
- Removed a stale `listing_id` assumption that caused the public indexer-status
  endpoint to silently omit resale projection health.

### Product reliability and provenance

- Completed chain-scoped projection verification across public artwork
  surfaces.
- Added indexed Creator, First Collector and Owner provenance on the artwork
  detail page, including the ownership timeline.
- Added creator public-nickname attribution to compact artwork cards with a
  shortened-address fallback and no per-card request fan-out.
- Corrected the profile Owned filter so a creator who later buys back their NFT
  appears in both the Created and Owned views.
- Added live-auction countdown pills to shared cards using projected end times
  and one local timer, with zero additional API or RPC requests.

### Safer user actions

- Audited lifecycle action visibility and submit-time authorization across
  publish, auction, settlement, resale and moderation surfaces.
- Enforced Base Sepolia-only artwork writes and live provider-account checks at
  the point of resale actions.
- Added a bounded post-publish confirmation and exact-artwork redirect without
  allowing a duplicate auction submission.

### Moderation foundations

- Added a 15-minute passkey step-up foundation for staff moderation behind a
  disabled feature flag.
- Added wallet-authenticated artwork report intake behind a separate disabled
  feature flag.
- Protected complaint records with forced RLS, service-role-only access,
  append-only submission evidence, concurrent deduplication and an approved
  five-new-reports-per-wallet rolling 24-hour bound.
- Report volume never hides an artwork automatically; a valid claim and an
  authorized review decision remain required.
